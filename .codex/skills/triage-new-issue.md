## Skill: Triage a New GitHub Issue

### When to use
A new issue is opened with a problem report.

### Steps
1. Identify the problem category:
   - Build/CI
   - Runtime error
   - Packaging
   - API usage
   - Documentation
2. Check for missing info:
   - OS
   - Versions (Flet, Python, Flutter)
   - Logs / stack traces
3. Ask for missing info if needed
4. Check for duplicates or related issues
5. Label or suggest labels (bug, question, docs, enhancement)
6. Suggest immediate next steps
